# AI Banking Document Intelligence & RAG System

An enterprise-grade, full-stack **Banking Document Intelligence System** built with **React**, **TypeScript**, **Node.js/Express**, and **Gemini 3.5 Flash** models to parse, extract, audit, and run multi-document **Retrieval-Augmented Generation (RAG)** queries over complex financial statements, loan agreements, and KYC documents.

---

## 🚀 Key Features

### 1. Multi-Document Ingestion & Native AI OCR
- Upload multiple bank statements, credit card bills, tax papers, and identity PDFs simultaneously.
- Automatic document classification into categories (e.g., *Bank Statement, Loan Document, KYC, Tax*).
- Multi-modal extraction using server-side Gemini 3.5 Flash, handling complex text and tables.

### 2. Live Vector Database Search Playground
- In-memory vector database matching cosine similarity embeddings generated by the `'gemini-embedding-2-preview'` model.
- Toggle between **Hybrid Search**, **Semantic Search**, and **Keyword Overlap** matches, with live retrieval score breakdowns.

### 3. Hallucination-Free Grounded RAG Chatbot
- Asks complex questions: *"What is my APR?", "List all high-value debits", "Is there any cash flow anomaly?"*
- Grounded answers generated strictly using fetched chunks context to prevent hallucination.
- **Source Citation Mapping**: Complete citation links highlighting document origins, page numbers, and vector confidence scores.

### 4. Interactive Plotly-Style Visual Dashboard
- Beautiful metrics dashboard rendered with **Recharts** representing cash flow inflows vs. outflows, monthly balance curves, category spend slices, and merchant spend analysis.
- Instant scoping filters to narrow down analytical views to individual statements or aggregated records.

### 5. Automated Fraud Detection & Security Scorecard
- Built-in compliance auditors checking duplicate charges (within 48 hours), suspicious cash outlays, large-value withdrawals, and atypical merchant categories.
- Real-time Fraud Risk Score (0-100) and structured safety recommendation grids.

### 6. Side-by-Side Document Comparison
- Multi-document comparison (e.g. month-over-month bank statements) outlining ending balances deltas, cash flow margins, category spend variance, and structured executive audits.

### 7. Session Protection & Exports
- Built-in secure audit profile login session management.
- Complete client-side exports to **CSV**, **JSON**, or custom print-formatted **PDF Audit Reports**.

## 🛠️ Tech Stack & Architecture

```
User → Browser Preview (Vite/React Dashboard)
            ↓  (API Requests / JSON Payload)
      Express Server (Custom TypeScript Entry Server)
            ↓  (Node.js / SDK Calls)
      @google/genai (Gemini 3.5 Flash & Gemini Embeddings)
```

- **Frontend**: React 19, TypeScript, Tailwind CSS v4, Motion, Lucide, Recharts
- **Backend**: Express, TSX, Esbuild (bundles server down to single efficient CJS file)
- **AI Models**: Google Gemini 3.5 Flash (`'gemini-3.5-flash'`), Gemini Embeddings (`'gemini-embedding-2-preview'`)

## 📦 Local Installation & Setup

1. **Clone & Install Dependencies**:
   ```bash
   npm install
   ```

2. **Configure Environment Variables**:
   Create a `.env` file at the root:
   ```env
   GEMINI_API_KEY="YOUR_GOOGLE_AI_STUDIO_API_KEY"
   APP_URL="http://localhost:3000"
   ```

3. **Launch Dev Server**:
   ```bash
   npm run dev
   ```

4. **Production Build**:
   ```bash
   npm run build
   npm start
   ```

## 💬 Core Technical Interview Questions Met

1. **Q: How does this system prevent LLM hallucinations during banking queries?**  
   *A: We apply a double-layer grounding system. First, the user's query is vectorized to retrieve top $K$ relevant chunks using cosine similarity. We then pass these chunks into Gemini with strict system instructions stating that the model must answer only from the retrieved context, returning specific citation maps, or state that the answer is not present.*

2. **Q: What is the benefit of the Hybrid Search over pure Semantic Search?**  
   *A: Hybrid search combines keyword match scores (which are excellent for exact account IDs, dates, and currency figures) with dense vector embeddings (which understand general concepts like "rent", "interest rates", or "fraud"). This combines the precision of lexical matching with the contextual intelligence of neural semantic search.*

3. **Q: How does this full-stack Express + Vite architecture optimize production deployment?**  
   *A: By compiling TypeScript directly on the server with `tsx` during dev, and bundling the Express API endpoints along with Vite's compiled static dist files into a unified `dist/server.cjs` file with `esbuild`, we ensure fast startup, low cold-starts, and zero dependency issues on modern container services like Cloud Run.*
